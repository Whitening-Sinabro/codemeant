## A. 문제 & 솔루션

### 한 줄 정의
> 초보 바이브코더를 위한 프로젝트 가치 분석 도구로, "이거 얼마에 팔 수 있어?"에 답한다.

### 고객 문제
1. 바이브코딩으로 만든 프로젝트의 시장 가치를 모름
2. 어디서, 어떻게 팔아야 할지 모름
3. 한국엔 Kmong 외에 판매 채널이 없음

### 우리 솔루션
- 로컬 에이전트로 프로젝트 메타데이터 추출 (코드 유출 걱정 없음)
- AI 팀(GPT + Gemini + Claude)이 가격 추정 + 판매 전략 제공
- 기존 플랫폼(Kmong, Gumroad) 연결

### 검증 상태
| 검증 유형 | 상태 | 증거 |
|-----------|------|------|
| 문제 검증 | ⏳ | 인터뷰 필요 |
| 솔루션 검증 | ⏳ | 인터뷰 필요 |
| 가격 검증 | ⏳ | 인터뷰 필요 |

└──────────────────┬──────────────────────────────────┘
                   │ (메타데이터만)
┌──────────────────▼──────────────────────────────────┐
│  CodeMeant API                                      │
│  ├─ GPT-4o-mini  ──┐                                │
│  ├─ Gemini 3 Flash ├─→ Claude Haiku (종합)          │

                   │
┌──────────────────▼──────────────────────────────────┐

└─────────────────────────────────────────────────────┘

## Acceptance Criteria

### MVP 완료 조건 (Definition of Done)

#### F001: 로컬 에이전트
- [ ] CLI 명령어로 프로젝트 폴더 지정 가능
- [ ] 메타데이터만 추출 (코드 내용 제외)
- [ ] 추출된 데이터가 5-10K 토큰 이내
- [ ] Windows/Mac/Linux 동작 확인

#### F002: 메타데이터 추출
- [ ] package.json/requirements.txt 등 의존성 파싱
- [ ] 파일 구조 트리 생성 (depth 3)
- [ ] README.md 내용 포함
- [ ] .gitignore 패턴 존중

#### F003: AI 분석
- [ ] GPT-4o-mini API 호출 성공
- [ ] Gemini Flash API 호출 성공
- [ ] Claude Haiku로 결과 종합
- [ ] 에러 시 graceful fallback

#### F004: 가격 추정
- [ ] 최소/최대/추천 가격 3가지 제시
- [ ] 각 가격에 대한 근거 포함
- [ ] 유사 프로젝트 예시 1개 이상

#### F005: 판매 전략
- [ ] 추천 플랫폼 2개 이상 제시
- [ ] 각 플랫폼별 장단점 설명
- [ ] 타겟 고객 정의 포함

#### F006: 크레딧 시스템
- [ ] Supabase에 users/credits 테이블 생성
- [ ] 분석 1회 = 5 크레딧 차감
- [ ] 잔액 부족 시 분석 차단
- [ ] Polar 결제 연동 (테스트 모드)

### 전체 MVP 완료 조건
- [ ] 위 F001-F006 모든 체크박스 완료
- [ ] E2E 테스트: 폴더 → 분석 → 결과 확인 플로우
- [ ] 에러 핸들링: 네트워크 오류, API 실패 처리
- [ ] 로딩 상태 UI 표시
